# 2D Link 1.6.3 — 用户指南

[产品主页](../README_zh-CN.md) · [快速入门](QUICK_START_zh-CN.md) · [兼容性](COMPATIBILITY_AND_PURCHASE_CHECKLIST_zh-CN.md) · [技术支持](SUPPORT_zh-CN.md) · [English](USER_GUIDE.md)

本指南说明 2D Link 1.6.3 中全部已公开功能的具体用法。如果你还没有完成一次手动贴图刷新和一次 Merge Layers 写回，请先阅读[快速入门](QUICK_START_zh-CN.md)。

插件在 Blender 中显示为 **Brandy 2D Link**，位于 3D 视图侧栏的 **Brandy** 标签中。

## 1. 理解项目工作流

一个 2D Link 项目会连接一个 Blender 集合、一组彼此独立的贴图文件，以及一个关联 PSD/PSB。

创建项目时，2D Link 会：

1. 检查合格的精灵面片及其贴图；
2. 将合格贴图复制到新项目目录；
3. 让 Blender 改用复制后的项目贴图；
4. 创建包含完整组装资产的关联 PSD/PSB。

创建项目前使用的文件称为**原始贴图**，创建项目时不会修改。项目目录中的副本称为**项目贴图**。单贴图编辑、手动刷新、自动刷新和 Merge Layers 写回都针对项目贴图。

界面按钮 **将 PS 合并组应用到源贴图** 中的“源贴图”同样指项目贴图，不会写回创建项目前的原始文件。

日常编辑有两条路径：

- **单贴图快速编辑**：直接打开一张项目贴图进行修改。
- **完整资产绘制**：打开关联 PSD/PSB，在完整资产中绘制，再把完成的图层明确写入一张或多张项目贴图。

两种方式都以保存为触发条件。2D Link 不会在绘制过程中逐笔传输。

## 2. 安装并打开插件

1. 保持购买得到的产品 ZIP 完整。
2. 在 Blender 中打开 **编辑 > 偏好设置 > 获取扩展** 或 **扩展**。
3. 打开右上角菜单，选择 **从磁盘安装**。
4. 选择完整的产品 ZIP。
5. 启用 **Brandy 2D Link**。
6. 打开 3D 视图，按 `N`，选择 **Brandy** 标签。

面板标题应显示 **Brandy 2D Link v1.6.3**。

不要单独安装 Python 或 JSX 文件，也不要把 GitHub 仓库 ZIP 当作插件包安装。

界面语言可在 **Photoshop 更多设置** 中切换为自动、English 或中文。修改后按照面板提示重新加载插件或重启 Blender。

## 3. 准备 Blender 资产

### 集合范围

将 **项目集合** 设置为代表一个资产的集合。目标部件位于嵌套集合中时，启用 **包含子集合**。

无关对象应放在所选范围之外。集合越清晰，项目验证和项目切换越容易理解。

### 部件结构

每个目标精灵部件应使用：

- 只有一个矩形外边界的平面网格；
- 矩形活动 UV；
- 与其他项目部件一致的 2D 平面朝向；
- 唯一的基础名称；
- 本地文件式 PNG、TGA、JPG 或 JPEG 贴图。

只要外边界保持矩形，可以使用内部细分。完整的矩形 UV 可以整体旋转或镜像，但主项目工作流不支持局部拉伸、UV 接缝、分离岛或重新排列的 UV 区域。

可以使用深度偏移叠放部件。动画、驱动、约束和修改器不会被传入关联文档；项目记录资产当前的基础构图。

### 名称

部件名称用于项目匹配和 Merge Layers 写回。匹配时会忽略 Blender 标准的三位数字复制后缀，例如 `.001`。移除后缀后，每个目标部件的名称仍必须唯一。

创建项目前应确定稳定名称。不要通过重命名生成项目文件来弥补名称问题；需要时应修正 Blender 对象名称并重新创建项目。

### 材质与贴图

材质顶层节点树中需要存在文件式 Image Texture。Image Texture 直接连接 Principled BSDF 的 Base Color 是最清晰的结构。

一个材质包含多个 Image Texture 时，插件必须能够识别其中一张主要项目贴图。受支持的导入记录或清晰的 Base Color 直连可以用于识别。任意自定义节点组内部隐藏的图像来源不属于主工作流。

创建项目前：

1. 保存 `.blend` 文件；
2. 将所有目标贴图保存为本地文件；
3. 完成需要的画布尺寸调整；
4. 解包存储在 `.blend` 内的图像；
5. 为重要生产作品保留独立备份。

创建项目后，不要改变项目贴图的像素宽度或高度。尺寸变化会使记录的布局失效，因此会被阻断。

## 4. 配置并检测 Photoshop

展开 **Photoshop 绑定**。

1. 将 **PS 启动路径** 设置为希望 2D Link 使用的 Photoshop 应用程序。
2. 启动已配置的 Photoshop，或点击 **打开 PS**。
3. 等待 Photoshop 完全启动。
4. 关闭欢迎页、保存对话框和其他模态窗口。
5. 点击 **检测 PS**。

检测成功后，面板会显示连接就绪和实际响应的 Photoshop 版本。安装了多个 Photoshop 版本时，检测前请关闭其他正在运行的版本。已配置的路径和 **检测 PS** 返回的版本比快捷方式名称或安装目录标签更可靠。

**PS 调用方式** 位于 **Photoshop 更多设置**。正常 Windows 工作流保持默认的 **自动选择**。

修改 Photoshop 启动路径、修改调用方式、改用另一个 Photoshop 版本，或面板提示检测结果已过期时，应重新执行 **检测 PS**。

### 手动执行模式

只有在自动 Photoshop 脚本调用被阻止，但 Photoshop 仍能执行本地 JSX 时，才使用 **手动执行**。

1. 展开 **Photoshop 更多设置**。
2. 将 **PS 调用方式** 设置为 **手动执行**。
3. 重新执行目标操作。
4. 按照 Blender 中显示的路径找到生成的 JSX 文件。
5. 在 Photoshop 中选择 **文件 > 脚本 > 浏览**，执行该文件。
6. Photoshop 完成后返回 Blender。

手动执行只改变任务启动方式，不会取消项目、文档、命名和文件状态要求。

## 5. 创建新项目

1. 设置 **项目集合**，并根据需要启用 **包含子集合**。
2. 在 **项目操作** 中选择 **新建项目到路径**。
3. 将 **项目包路径** 设置为一个空的本地目录，或一个尚不存在的路径。
4. 只有需要在完整资产外增加透明工作区域时，才调整 **工作画布边距**。
5. 确认 **检测 PS** 已就绪。
6. 点击 **创建（需 PS 正常）**。
7. 在任务完成前保持 Blender 与 Photoshop 打开。

**工作画布边距** 只会在完整关联文档外增加透明区域，不会改变单张贴图尺寸，也不会改变部件之间的相对位置。

创建过程中，插件会检查集合、部件、UV、图像文件、名称和构图。成功后，**Photoshop 绑定** 与 **Photoshop 互通** 区域会显示当前项目和关联文档。

不要手动重命名、替换或移动单个生成文件。应始终保持完整项目目录在一起。

创建停止时，打开 **执行报告**，修正第一个被阻断的条件，再重新创建。在报告明确说明之前，应保留未完成项目目录，以便判断是恢复还是删除。

## 6. 使用当前项目

项目完成绑定后，**Photoshop 互通** 区域会显示当前项目与关联文档，并提供日常操作按钮。**源贴图状态** 会显示当前项目是否能够被监测和刷新；状态暂停或被阻断时，请按照其下方提示处理。

### 打开关联文档

点击 **用 PS 打开关联文档**，打开完整资产 PSD/PSB。

### 打开一张贴图

将一个项目精灵设为活动对象，点击 **用 PS 编辑活动贴图**。插件会打开该部件对应的主要项目贴图。

### 手动刷新项目贴图

点击 **手动刷新**，重新载入当前项目内受支持的图像。自动刷新关闭时，或希望立即检查 Photoshop 保存结果时使用。

如果受影响的 Blender Image 存在未保存像素修改、已打包进 `.blend`、不再是文件式图像，或无法安全更新，刷新会被拒绝。应阅读提示和执行报告，而不是强制重新载入。

### 自动刷新

手动刷新成功一次后可以启用 **自动刷新**。插件会监测当前项目中受支持的贴图文件，并在确认保存完成且文件稳定后重新载入图像。

自动刷新只监测当前项目。在项目未完成、正在验证项目切换、Photoshop 正在写回，或文件不再匹配项目记录时，它可能暂停。

保存后贴图没有更新时：

1. 确认编辑的是当前项目 `textures` 目录中的文件；
2. 确认 Photoshop 没有改名或另存到其他位置；
3. 确认宽度和高度没有变化；
4. 等待 Photoshop 完成写入；
5. 执行 **手动刷新**；
6. 如果刷新被阻断，阅读 **执行报告**。

## 7. 直接编辑单张贴图

修改只影响一张贴图，或需要精确控制像素时，使用直接编辑。

1. 在 Blender 中将目标精灵面片设为活动对象。
2. 点击 **用 PS 编辑活动贴图**。
3. 在 Photoshop 中绘制。
4. 原位保存同一文件。
5. 返回 Blender，执行 **手动刷新**，或让自动刷新检测保存完成。

往返过程中不要改变文件名、目录、格式、宽度或高度。需要切换到另一种已存在的文件格式时，应使用后文的 **重读贴图格式**，不要直接另存为其他文件。

精细清理以及跨重叠部件的柔和透明边缘，建议直接编辑单张贴图，因为操作对象就是准确的目标文件，不需要额外分发。

## 8. 使用关联 PSD/PSB

关联文档包含三个保留的根级组。

### Brandy | Linked Content

该组包含项目各张贴图对应的生成关联智能对象，其名称也是可用的 Merge Layers 目标名称。

不要重命名、移动、复制、变换、翻转、扭曲、重新关联、替换或重组这些智能对象，也不要重命名或移动保留组。

### Brandy | Merge Layers

需要写回项目贴图的完成绘制内容放在这里。只有直接位于组内的可见绘制图层会被处理，子组不会作为写回目标。

该组保持 100% 不透明度，混合模式使用 Normal 或 Pass Through。

### Brandy | Merged Layers

成功写回后，已处理的绘制图层会移动到该组。这样可以清空当前 Merge Layers 组，同时把绘制内容保留在关联文档中。

参考图、备注、草图和普通工作图层可以放在三个保留组之外。保存 PSD/PSB 只会保存工作文档，不会自动更新全部项目贴图。

## 9. 将 Merge Layers 应用到项目贴图

需要在完整资产中绘制，再将完成内容分发到一张或多张贴图时，使用 Merge Layers。

### 准备目标图层

对于每张目标贴图：

1. 在 **Brandy | Merge Layers** 内直接创建或粘贴一张可见绘制图层；
2. 从 **Brandy | Linked Content** 复制准确的目标名称；
3. 将该名称原样用于绘制图层，注意大小写；
4. 保持图层直接位于组内，不要放入子组。

一项绘制内容跨越多个部件时，为每个目标复制一份，并分别使用对应目标名称。写回时，超出各目标贴图画布的像素会被裁切。

执行前：

- 保存关联 PSD/PSB；
- 保持三个保留组和生成智能对象不变；
- 保持所有受影响项目贴图的记录尺寸；
- 如果受影响的项目贴图已在 Photoshop 中打开，先保存，并确保文档只有一张绘制图层且没有图层组。

### 颜色模式与文件格式检查

RGB 8 位目标文档可以直接使用。

对于受支持的 8 位或 16 位 RGB、灰度、CMYK 或 Lab 文档，Photoshop 可能请求转换为 RGB 8 位。继续前应阅读提示，因为转换可能改变颜色或精度。

不支持的模式或位深必须先手动转换。缺少色彩配置或使用非 sRGB 配置会出现警告，因为 Blender 通常把这些贴图作为 sRGB 输入读取。

JPG 和 JPEG 目标会额外要求确认，因为保存为有损压缩。需要反复编辑时尽量使用 PNG 或 TGA。

### 执行写回

1. 返回 Blender。
2. 确认当前项目正确。
3. 点击 **将 PS 合并组应用到源贴图**。
4. 阅读并确认可能出现的警告。
5. 等待 Photoshop 与 Blender 完成。
6. 如果图层被跳过或图像没有刷新，阅读执行报告。

修改文件前，2D Link 会检查项目、关联文档、保留组结构、目标名称、目标尺寸和当前文件状态，并为受影响的项目贴图创建经过验证的备份。

成功后：

- 匹配的项目贴图会更新；
- 已处理图层会移入 **Brandy | Merged Layers**；
- 关联文档会保存；
- Blender 会尝试重新载入受影响图像。

只有报告提示 Blender 图像未刷新，或可见结果仍旧时，才需要点击 **手动刷新**。

## 10. 撤销最近一次 Merge Layers 写回

**撤销上一次合并** 会恢复最近一次成功 Merge Layers 操作改动的项目贴图。

发现结果不正确时应立即使用，不要先编辑或保存受影响贴图。

撤销会确认当前文件仍与写回后的记录状态一致。如果目标文件再次发生变化、恢复记录缺失，或项目已不匹配，命令会被拒绝，以免覆盖更新的内容。

该功能只提供一次项目级撤销，不是历史记录浏览器。生产工作仍应保留正常备份和版本管理。

## 11. 绑定或切换已有项目

该工作流用于重新打开项目、完整移动项目目录后重新连接，或在兼容的项目变体之间切换同一资产集合。

1. 将 **项目集合** 设置为需要使用该项目的 Blender 集合。
2. 在 **项目操作** 中选择 **绑定或切换已有项目**。
3. 将 **项目包路径** 设置为已有的 2D Link 项目目录。
4. 出现 **准备切换** 时检查当前项目与候选项目。
5. 确认 **检测 PS** 已就绪。
6. 点击 **绑定或切换（需 PS 正常）**。
7. 等待验证完成。

候选项目通过验证前，当前项目会保持活动状态。部件身份、构图和贴图尺寸必须与所选 Blender 集合匹配。

可以将完整项目目录移动到另一个本地位置，但内部结构必须保持不变。选择新位置的项目目录，再执行 **绑定或切换（需 PS 正常）**。不要单独移动生成贴图、元数据文件或关联 PSD/PSB。

切换成功后，自动刷新保持关闭。先执行一次 **手动刷新** 并确认结果，再根据需要启用自动刷新。

## 12. Photoshop 任务控制

大多数 Photoshop 任务都应等待自然完成。创建项目、切换项目、应用 Merge Layers 或撤销上一次合并时，不要关闭 Blender 或 Photoshop。

### PS 通信等待时限（秒）

该设置控制 Blender 连续等待多长时间后改为定期检查结果。达到时限不会停止 Photoshop，也不代表操作失败。

### 请求中断

希望活动任务在安全位置停止时使用 **请求中断**。中断需要配合完成，Photoshop 可能仍需结束当前步骤并恢复文件。

### 停止等待

Photoshop 仍在运行，或 Blender 暂时无法读取最终状态时，可使用 **停止等待** 结束 Blender 的前台等待。该操作不会取消 Photoshop。在明确最终状态前，应保持两个应用程序和项目文件不变。

### 确认任务已经结束

Blender 无法判断任务是否仍在运行时会显示该按钮。只有在确认 Photoshop 已结束并且相关文件不再写入后才使用。该操作只会清除未知任务状态，不会完成或撤销未结束的 Photoshop 工作。

## 13. 执行报告、任务记录与恢复

### 执行报告

任务停止或结果不完整时，展开 **Photoshop 更多设置**，点击 **打开执行报告**。最近一次完整报告也保存在 Blender 的以下文本数据块中：

`BRANDY_2D_LINK_Last_Report`

先处理第一个被阻断的条件，再重新执行。后续信息通常只是同一个初始问题带来的结果。

### 打开任务记录目录

只有需要更深入诊断，或维护者要求时才使用 **打开任务记录目录**。记录可能包含本地路径、账户名、项目名和作品引用，分享前必须检查并删除隐私内容。

### 恢复未完成项目

项目创建被中断并留下可恢复标记时，会出现此按钮。

1. 保持未完成项目目录不变。
2. 确认已配置的 Photoshop 版本连接就绪。
3. 关闭无关 Photoshop 文档或对话框。
4. 只有面板或报告要求时，才点击 **恢复未完成项目**。

恢复被拒绝时，应根据新的报告处理，不要手动修改生成文件。

### 恢复中断写回的源贴图

只有 Merge Layers 写回被中断，并且面板或报告识别到可恢复备份状态时才使用。

1. 完全退出 Photoshop，避免它继续保存目标文件。
2. 保持项目目录不变。
3. 点击 **恢复中断写回的源贴图**。
4. 恢复后执行 **手动刷新**，确认结果。

任务备份不会自动删除。在项目完成检查前应继续保留。

### 修复项目部件标识

只有项目绑定报告明确指出两份项目记录仅在部件标识上不一致时，才使用 **修复项目部件标识**。该操作会创建备份并修复这一特定问题，不用于修复重命名部件、改变构图或移动文件。

### 清除失效项目锁

异常退出后可能留下项目锁。只有确认所有使用该项目的 Blender 与 Photoshop 操作都已结束，并且没有进程继续写入项目文件时，才能清除。

面板提示项目由另一个 Blender 实例持有时，应返回该实例或等待其任务完成，不要强制清除活动锁。

## 14. PhotoshopToSpine 导入

该工具导入受支持的静态 PhotoshopToSpine JSON 结构，并在 Blender 中创建带贴图的精灵面片，不会导入动画。

1. 展开 **PhotoshopToSpine 导入**。
2. 将 **JSON 文件** 设置为源 JSON。
3. 将 **图片格式** 设置为源图片使用的扩展名：PNG、JPG 或 TGA。
4. 贴图包含透明度时启用 **使用 Alpha**。
5. 只有源图片使用预乘 Alpha 时才启用 **预乘 Alpha**。
6. 需要镜像完整导入构图时启用 **水平镜像**。
7. 使用 **导入缩放** 将源像素坐标转换为 Blender 单位，默认值为 `0.001`。
8. 使用 **面片 Z 轴间距** 控制导入部件的深度间隔，默认值为 `0.001`。
9. 点击 **导入 PhotoshopToSpine JSON**。

导入器要求完整的静态结构和匹配图片。验证失败时会在留下不完整资产之前停止。修正执行报告中的第一个错误后重新导入。

创建的集合和根控制器会保持导入构图。之后可能需要使用 **为选中项恢复导入材质** 时，应保留原始 JSON 与图片目录。

## 15. PhotoshopToSpine 导出

该工具把当前 Blender 中合格的精灵构图导出为 PhotoshopToSpine 兼容的静态 JSON，不会创建、绑定或修改 Photoshop 项目。

1. 将 **项目集合** 设置为需要导出的集合。
2. 根据需要启用 **包含子集合**。
3. 展开 **PhotoshopToSpine 导出**。
4. 将 **输出路径** 设置为 JSON 文件或输出目录。
5. 点击 **从项目集合导出 JSON**。

导出会检查精灵名称、图片路径、矩形结构、UV、构图和显示比例一致性。必要部件未通过验证时，不会写出不完整 JSON。根据执行报告修正资产后重新导出。

## 16. Spine2D 静态导入

该工具从受支持的 Spine Region Attachment 导入有限的静态 Setup Pose，不是完整 Spine 导入器。

1. 展开 **Spine2D 导入（有限）**。
2. 设置 **JSON 文件**。
3. 选择 **图片格式**。
4. 根据源图片设置 **使用 Alpha**、**预乘 Alpha** 和 **水平镜像**。
5. 除非可信 JSON 明确指向 JSON 目录之外的图片目录，否则保持 **允许外部贴图目录** 关闭。
6. 只有受支持的 Region Attachment 缺少宽度或高度时，才设置 **缺失尺寸补齐**，默认值为 100 px。
7. 设置 **导入缩放** 与 **面片 Z 轴间距**。
8. 点击 **导入静态 Spine2D JSON**。

只会导入受支持的静态 Region Attachment。完整骨架、动画时间线、Mesh Attachment、Region Sequence、不支持的约束、双色 Tint，以及不支持的混合或变换行为都会被拒绝。

图片路径必须能在允许目录中唯一解析。如果多个文件可能匹配同一名称，或路径越过允许边界，导入器会停止而不是猜测。只有信任 JSON 来源时才允许访问外部图片目录。

## 17. 便捷功能

### 重读贴图格式

该功能让选中的精灵改用基础名称相同、扩展名不同的现有文件。

1. 预先准备替换文件，保持相同基础名称，并使用目标 PNG、TGA 或 JPG 扩展名。
2. 选择需要更新的精灵对象。
3. 在 **便捷功能** 中设置 **刷新格式**。
4. 点击 **重读贴图格式**。
5. 查看执行报告中被跳过的对象。

该工具不会转换图像数据，也不会创建替换文件，只会切换到已经存在的同名文件。打包、未保存、共享或其他不安全状态的图像可能被跳过。

### 复制着色器参数到选中项

该功能把一个精灵材质中兼容、未连接的 Principled BSDF 输入值复制到其他选中精灵，同时保留各自贴图连接。

1. 先选择目标对象。
2. 最后选择源对象，使其成为活动对象。
3. 点击 **复制着色器参数到选中项**。
4. 如果结果为部分完成，查看报告。

只有名称相同，并且连接输入结构兼容的 Principled BSDF 参数会被复制。已连接输入会继续保持连接。该工具不会重建无关的自定义节点系统。

### 为选中项的材质查重与合并

选择目标精灵对象，点击 **为选中项的材质查重与合并**。

只有能够安全共享、由插件创建且未被修改的匹配材质会被合并。工具会识别标准 `.001` 类重复项。自定义或经过修改的材质不会仅凭名称被合并。

### 为选中项恢复导入材质

JSON 导入创建的精灵，其第一个材质槽被移除或修改时使用。

1. 选择一个或多个导入精灵。
2. 点击 **为选中项恢复导入材质**。
3. 阅读可能出现的共享网格或外部贴图目录提示。
4. 只有信任源 JSON 与图片位置时才确认。

工具会根据导入时存储在精灵上的信息重建第一个材质槽。原始 JSON 已移动时，如果当前选择的 JSON 与导入资产匹配，可以作为备用来源。

### 独显选中项

选择需要检查的对象，在 3D 视图中点击 **独显选中项**。再次点击同一按钮可退出局部视图，并恢复之前的选择状态。

必须从 3D 视图执行。如果已经处于其他局部视图状态，请先退出，再使用插件的独显切换。

## 18. 实际排查顺序

正常操作没有完成时：

1. 阅读可见提示；
2. 打开 **执行报告**；
3. 修正第一个被阻断的条件；
4. 确认目标 **项目集合**、当前项目和项目目录正确；
5. Photoshop 路径、版本或调用方式发生变化时，重新执行 **检测 PS**；
6. 确认项目文件仍位于本地磁盘，并且贴图尺寸没有变化；
7. 问题可能与特定资产有关时，在项目副本中重新测试。

只有报告明确指出对应恢复状态时才使用恢复按钮。问题仍无法解决时，请按照[技术支持](SUPPORT_zh-CN.md)页面整理信息，并将已删除隐私内容的报告发送至 **brandyspe2026@gmail.com**。
